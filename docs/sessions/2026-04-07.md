## Bugs & Fixes

### 18. Download path $HOME resolved on container
**Problem**: `$HOME` in Node.js template literal resolved to `/root` (Alpine container) not `/Users/admin` (Mac)
**Solution**: Use `os.path.expanduser("~/.exo/models/...")` in Python script

### 19. Download source was `settings.source` (Model Manager legacy)
**Problem**: `_startDownload` referenced `settings.source.paths.exo` which doesn't exist in ExoScopy
**Solution**: Use `settings.nodes[0]` as download target

### 20. Variable name conflict — white screen
**Problem**: `models` declared twice in ModelsPage (matrixData destructuring + useState)
**Solution**: Renamed to `chatModels`

### 21. useRef outside component — white screen
**Problem**: `useRef` used at module level for persistent search state
**Solution**: Replaced with plain object

### 22. HF hub check red on secondary nodes
**Problem**: Config check showed red for huggingface_hub on nodes that don't need it
**Solution**: Only check on first node, show "not required" in gray on others
